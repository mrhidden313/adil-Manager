## 1. VISION & MISSION
AK FLOW is a B2B SaaS platform designed to manage multiple tenant agencies. It allows the Super Admin to onboard "Agencies" (Companies), assign Managers to them, and let those Managers handle their own Sales and Fulfillment teams.
The core workflow: Sales agents submit "Pads" or "Tickets", and Fulfillment agents process and complete them.

## 2. TECH STACK & INFRASTRUCTURE
* **Frontend:** SvelteKit (Svelte 5), TailwindCSS v4, Vite PWA.
  * *Deployment:* Hosted on Vercel (`akflow.vercel.app`).
  * *Architecture:* Pure Single Page Application (SPA). Server-Side Rendering (SSR) is completely DISABLED to prevent `sessionStorage` errors.
* **Backend:** Node.js, Express, TypeScript, Prisma ORM.
  * *Deployment:* Hosted on an Ubuntu VPS (IP: `69.164.245.230`) using PM2 (process manager).
  * *Web Server:* Nginx acts as a reverse proxy, routing traffic from Port 443 (HTTPS) to Port 3000 (Node.js).
  * *SSL:* Let's Encrypt / Certbot is used for the domain `akflow.duckdns.org`.
* **Database:** PostgreSQL (Hosted on Supabase).

## 3. AUTHENTICATION & SECURITY
* **System:** Custom JWT-based stateless authentication.
* **Roles:** SUPER_ADMIN, MANAGER, SALES, FULFILLMENT.
* **Super Admin Login:** adminadil@akflow.com / adill1122@@@
* **Critical Security Patch:** The JWT middleware (`auth.middleware.ts`) was specifically modified to query the database on every request to ensure the user still exists and is `isActive`. This was done to prevent deleted users from retaining access via unexpired JWT tokens.

## 4. CRITICAL PROBLEMS FACED & SOLVED
If you are an AI reading this, DO NOT revert these fixes:
1. **Vercel 500 Internal Server Errors:** The frontend heavily relies on `sessionStorage` and browser APIs. During early deployments, Vercel attempted to render pages on the server (SSR), causing massive 500 crashes because `sessionStorage` is undefined in Node.js. 
   * *Solution:* We created `src/routes/+layout.ts` with `export const ssr = false;`. DO NOT re-enable SSR.
2. **Missing Imports in Svelte:** Un-imported capitalized components (like `<BottomNav>`) cause Vercel builds to crash at runtime. Always ensure every component is explicitly imported.
3. **VPS Nginx Configuration:** The backend requires HTTPS to communicate with Vercel. We used DuckDNS (`akflow.duckdns.org`) and Nginx to bridge this. The proxy headers `X-Forwarded-Proto` and WebSockets `Upgrade` headers are required in the Nginx config for this to work smoothly.
4. **Database Resets:** The project relies on `src/seed.ts` to create the initial Super Admin. To wipe the database safely, we use `npx prisma db push --force-reset` and then run the seed script.

## 5. REPOSITORY STRUCTURE
* `/frontend/` - Contains all SvelteKit code.
  * `.env` contains `VITE_API_URL=https://akflow.duckdns.org`.
* `/backend/` - Contains all Node.js Express code.
  * `.env` contains `DATABASE_URL` (Supabase) and `JWT_SECRET`.
  * `src/routes/` and `src/controllers/` handle API logic.
  * `prisma/schema.prisma` holds the database schema.
